# Alexa+ Agent-Assisted DX

A framework visualizing how the Alexa+ orchestrator supports developers across the builder journey — from ideating customer moments to monitoring live integrations.

Built as a working prototype to support case study presentation and portfolio documentation.

## What's in here

- `index.html` — fully interactive table with collapsible row groups, no dependencies beyond Google Fonts

## Live

https://[your-github-username].github.io/alexa-builder-orchestration-experiences/

## Structure

The table maps four developer signposts against three layers of the orchestrator experience:

| Signpost | Description |
|---|---|
| DX01 · Ideate | Define customer moments before writing any code |
| DX02 · Build & Simulate | Build and preview interactions in one place |
| DX03 · Test & Deploy | Validate against edge cases and ship with confidence |
| DX04 · Monitor & Manage | Gain visibility into live customer usage |

**Row groups**
- **Developer Need** — JTBD statement per signpost
- **Where It Happens** — primary surface (Website or IDE) and key tasks
- **How It Works** — agent responsibilities and developer control mode
