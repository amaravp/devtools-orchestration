Developer Need What does the developer need at this signpost? ▾
JTBD When I'm exploring what's possible with Alexa+, I want to map out the customer moments my integration could own, so I can define a clear scope before writing a single line of code. When I'm developing my integration, I want to build and preview customer interactions in one place, so I can iterate quickly without switching between tools. How It Works What is the agent doing, and how much control does the developer have? ▾
Agent responsibilities Orchestrator

Surfaces integration type and drafts moment scaffold; proposes Core, Edge, and Boundary moments from URL or conversation

 Orchestrator Sub-agent

Delegates to sub-agents for capability mapping and routing validation; flags gaps before developer advances

 Sub-agent Judge agent

Sub-agents execute test runs autonomously; judge agent evaluates results and surfaces a go/no-go recommendation

 Orchestrator

Monitors continuously across all surfaces; surfaces anomalies, intent drift, and performance degradation for developer review
