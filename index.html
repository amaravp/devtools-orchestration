Developer Need What does the developer need at this signpost? ▾
Job to be done When I'm connecting my service to Alexa+, I want to map my API schema and generate a testable prototype quickly, so I can validate the experience before investing in full build-out. When my prototype is ready, I want to test, fix, promote, and certify my integration with confidence, so I can ship without manually managing every step. When my integration is live, I want real-time visibility into errors, usage, and global readiness, so I can catch degradation and scale before it impacts customers.

How It Works Where does collaboration end and autonomy begin? ▾
Human-in-the-loop
✦ Always
Developer confirms integration spec and scaffold before agent proceeds; no build begins without explicit approval
Selective
Developer makes the final go/no-go call before promotion and certification; agent handles test execution and bug fixes
On demand
Developer sets thresholds and reviews flagged issues; routine monitoring and rollback operations run without developer input
Agent responsibilities Orchestrator Sub-agent

Orchestrator surfaces integration type and delegates to sub-agents to scaffold and generate a testable prototype

 Judge agent Sub-agent

Sub-agents execute test runs and apply fixes autonomously; judge agent evaluates results and surfaces a go/no-go recommendation before promotion and certification

 Orchestrator Sub-agent

Orchestrator monitors continuously; sub-agents handle versioning, rollbacks, and locale validation without developer-initiated prompts
Specific agents Integration agent Prototyping agent

Integration agent maps API schema to SDK models; Prototyping agent generates a testable prototype from the integration spec

 Testing agent Bug fixing agent Promotion agent Certification agent

Testing agent runs simulated interactions; Bug fixing agent identifies and applies targeted fixes; Promotion agent pushes to target environment; Certification agent validates against compliance requirements

 Lifecycle ops agent Monitoring agent Internationalization agent

Lifecycle ops agent manages versioning and rollbacks; Monitoring agent surfaces real-time errors and usage patterns; Internationalization agent adapts content for new locales
